1. Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?

git checkout fileName.txt

2. Как вернуть файл из индекса в рабочий каталог?

git restore --staged fileName.txt

3. Как отменить ошибочный commit?

git reset HEAD^ или git reset HEAD~1 - откатиться на 1 шаг назад
git restore fileName - удалить commit

4. Как создать новую ветку и переключится в нее?
git branch name - создать ветку с именем name
git checkout name - переключиться на ветку name

5. В чем отличение между fast-forward merge  and non fast-forfard merge?
Их закономерность в том, что всегда при fast-forward merge (быстром слиянии) происходит новый коммит,
а приnon fast-forfard merge (небыстром слиянии) – сохраняется присутствие коммитов.

6.Когда может возникнуть конфликт слияния веток, как  его разрешить?
	
Конфликты слияния случаются, если обнаруживаются изменения в той же самой версии файла или если один пользователь изменяет файл, а это тот же файл обработки.

Открыть файл в любом редакторе, чтобы поосмотреть и исключить конфликтующий текст. Git имеет набор маркеров для указания конфликтующего текста. 
Например:
Текст между <<<<<<< и ======= Этот текст является собранием представителей конфликтующего.
Текст между ======= и >>>>>>> Является исходной ветви конфликтующего текста.

Нужно изменить файл, чтобы разрешить конфликтующий текст. Удаляя маркеры выявления слияния, вы сообщаете Git, что вы разрешили конфликт.
Запустить git add fileName, чтобы подготовить файл после разрешения всего конфликтующего текста.

7.Можно ли удалить ветку? Как? 

Ветку удалить можно.
Сначала выйти из этой ветки, затем ввести команду:
git branch -d name(имя ветки).
